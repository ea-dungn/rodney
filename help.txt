rodney - Chrome automation from the command line

USAGE:
  rodney <command> [arguments]

GETTING STARTED:
  rodney start                    # Launch browser
  rodney open example.com         # Navigate to a page
  rodney title                    # Get page title
  rodney stop                     # Shut down browser

BROWSER LIFECYCLE:
  start                           Launch headless Chrome
  stop                            Shut down Chrome
  status                          Show browser status and current page

NAVIGATION:
  open <url>                      Navigate to URL (http:// added if missing)
  back                            Go back in history
  forward                         Go forward in history
  reload [--hard]                  Reload current page (--hard bypasses cache)

PAGE INFORMATION:
  url                             Print current URL
  title                           Print page title
  html [selector]                 Print HTML (full page or element)
  text <selector>                 Print text content of element
  attr <selector> <name>          Print attribute value
  pdf [file]                      Save page as PDF (default: page.pdf)

INTERACTION:
  js <expression>                 Evaluate JavaScript expression
  click <selector>                Click an element
  input <selector> <text>         Type text into an input field
  clear <selector>                Clear an input field
  file <selector> <path|->        Set file on a file input (- for stdin)
  download <sel> [file|-]         Download href/src target (- for stdout)
  select <selector> <val>         Select dropdown option by value
  submit <selector>               Submit a form
  hover <selector>                Hover over an element
  focus <selector>                Focus an element

SCROLLING:
  scroll <selector>               Scroll element into view
  scroll up|down|left|right [px]  Scroll direction (default 300px)
  scroll top|bottom               Scroll to page top/bottom

KEYBOARD:
  key <Enter|Tab|Escape|...>      Press a named key
  key <ctrl+a|shift+Tab|...>      Press a key combination
  key <text>                      Type text character by character

WAITING:
  wait <selector>                 Wait for element to appear and be visible
  waitload                        Wait for page load event
  waitstable                      Wait for DOM to stop changing
  waitidle                        Wait for network to be idle
  waitfor [--timeout N] <js-expr> Wait until JS expression is truthy
  sleep <seconds>                 Sleep for N seconds

SCREENSHOTS:
  screenshot [-w N] [-h N] [file] Take full page screenshot
  screenshot-el <sel> [file]      Screenshot a specific element

TABS:
  pages                           List all open tabs (* marks active)
  page <index>                    Switch to tab by index
  newpage [url]                   Open a new tab
  closepage [index]               Close a tab (default: active tab)

ELEMENT QUERIES:
  exists <selector>               Check if element exists (exit 0/1)
  count <selector>                Count matching elements
  visible <selector>              Check if element is visible (exit 0/1)

ACCESSIBILITY TESTING:
  ax-tree [--depth N] [--json]    Dump accessibility tree
  ax-find [--name N] [--role R] [--json]
                                  Find accessible nodes by name/role
  ax-node <selector> [--json]     Show element's accessibility properties

CONSOLE:
  console [--errors] [--json]     Read captured console messages
  console --follow [--errors]     Stream console output (Ctrl+C to stop)
  console --clear                 Clear captured messages

PERFORMANCE:
  perf metrics [--json]           Show runtime performance metrics
  perf vitals [--json]            Show Core Web Vitals (LCP, CLS, TTFB)
  perf timing [--json]            Show navigation timing breakdown
  perf profile <secs> [file]      Record CPU profile (outputs .cpuprofile)
  perf trace <secs> [file]        Record browser trace (outputs JSON for Perfetto)

REACT PROFILING:
  react hook                      Install React DevTools hook (persists across navigations)
  react tree [--json]             Show React component tree
  react renders [--json]          Show render commits with per-component timing
  react flamegraph <file>         Export renders as speedscope flamegraph

  Workflow:
    rodney react hook                    # Install hook
    rodney open https://my-react-app.dev # Navigate
    rodney react tree                    # See component hierarchy
    rodney react renders                 # See what rendered and how long
    rodney react flamegraph renders.json # Export flamegraph
    npx speedscope renders.json          # Visualize

COOKIES:
  cookies list [--json]           List all cookies for current page
  cookies get <name>              Get a cookie value by name
  cookies set <name> <value> [--domain D] [--path P] [--secure] [--httponly]
                                  [--expires UNIX]
                                  Set a cookie
  cookies delete <name> [--domain D]
                                  Delete a cookie by name
  cookies clear                   Clear all cookies

STORAGE:
  storage list [--session] [--json]
                                  List localStorage items (--session for sessionStorage)
  storage get <key> [--session]   Get a storage value
  storage set <key> <value> [--session]
                                  Set a storage value
  storage delete <key> [--session]
                                  Delete a storage key
  storage clear [--session]       Clear all storage

NETWORK INSPECTION:
  network list [--json] [--filter <url-pattern>]
                                  List all network requests with timing info
  network filter <pattern>        Search response bodies (great for GraphQL!)
  network clear                   Clear network request log
  network save <file.json>        Save network log to JSON file

  Examples:
    rodney network list                    # Show all requests
    rodney network list --filter "/api/"   # Filter by URL
    rodney network filter "getUserProfile" # Find GraphQL operations
    rodney network save network.json       # Export for analysis

OPTIONS:
  --version                       Print version and exit
  --help, -h, help                Show this help message

ENVIRONMENT VARIABLES:
  ROD_CHROME_BIN                  Path to Chrome/Chromium binary
  ROD_TIMEOUT                     Command timeout in seconds (default: 30)
  ROD_SINGLE_PROCESS              Set to "1" to run Chrome in single-process
                                  mode (needed for gVisor/container environments)
  ROD_GPU                         Set to "1" to enable GPU acceleration and
                                  WebGPU (navigator.gpu). Requires hardware GPU
                                  or a machine with /dev/dri/ for SwiftShader.

EXAMPLES:
  # Basic automation
  rodney start
  rodney open https://example.com
  rodney click "button#submit"
  rodney waitload
  rodney screenshot result.png
  rodney stop

  # Extract data
  rodney open news.ycombinator.com
  rodney text ".title" | head -1

  # Debug API calls
  rodney open https://app.example.com
  rodney waitidle
  rodney network list --filter "/graphql"

  # Accessibility check
  rodney open mysite.com
  rodney ax-find --role button --json | jq 'length'
